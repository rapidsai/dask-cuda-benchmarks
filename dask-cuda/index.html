Overview

Some simple dask-cuda benchmarks, measuring throughput over time

Multi-node experiments

These are run on Draco, see the run scripts.

Merge throughput

This measures achieved throughput / GPU in a multi-node weak scaling experiment. Each worker has 40M rows, and the total processed data is ngpu * nrow * 2 * 8 * 2 bytes

Transpose sum

This measures achieved throughput / GPU in a multi-node strong scaling experiment. The benchmark transposes a distributed 2D array (of size 50000 x 50000), adds it to itself (pointwise parallel), before summing up all entries. The total processed data is nrow * ncol * 8 bytes.

Single-node experiments

These are run as part of the UCX-Py nightlies, which also contains historical performance for point-to-point message passing.

Merge throughput

This measures achieved throughput / GPU in a single-node experiment. Each worker has 100M rows, and the total processed data is ngpu * nrow * 2 * 8 * 2 bytes

Transpose sum

This measures achieved throughput / GPU in a single-nodes experiment. The benchmark transposes a distributed 2D array (of size 25000 x 25000), adds it to itself (pointwise parallel), before summing up all entries. The total processed data is nrow * ncol * 8 bytes.
